# CloudWatch

## Purpose

CloudWatch was enabled to provide visibility into metrics and support monitoring familiarity within the lab environment. While it was not the primary detection mechanism for the tested misconfigurations, it contributed to the overall monitoring layer.

## Role in the Project

CloudWatch supported operational awareness of the environment and helped round out the monitoring stack alongside CloudTrail and AWS Config.

## Notes

No advanced alarms or automated responses were configured in this baseline environment. The project primarily focused on configuration monitoring and activity logging rather than automated alerting.
